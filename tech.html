In this project I build a simple web application that allows users to simulate buying and selling stocks they "own". A small part of the web application is made available by Harvard's CS50x web programming track. I implemented the features that would allow a user to do the following

  1. Register. Create the app route to allow new users to register for an account. New user added into the SQL database.
  2. Quote. Allow user to search for a stock and get a quotation of its current price.
  3. Buy. Allow user to buy a number of shares. Transaction added into SQL database.
  4. Index. Summarises the shares of each stock, the value and the total cash the user has.
  5. Sell. Allow user to sell a number of shares. Transaction added into SQL database.
  6. History. Displays all the transaction history according to the transaction date. Shows the number of shares sold/bought and at what price.
  7. Sort transaction history. Allows the user to sort their transaction history in chronological order (or in reverse order).
The back-end is developed using Flask framework and jinja templating language. The front-end is developed using HTML, CSS and Bootstrap4. The database is managed by SQLite3. View the code I have written for this project here.

This is the final project done in fulfilment of Harvard's CS50x online course. I aim to create a full-stack application. Final Year Project (Aug 2019 - April 2020)
The aim of my project was to develop a machine learning based surrogate model to predict the surface temperature of surfaces found in an urban environment. This model would serve as a quick and accurate tool for building developers to quantify the changes to surface temperature based on their design choices. The machine learning algorithm I was used was support vector regression due to its reported efficiency on small datasets.

In this project, I gained proficiency in the full machine learning pipeline which consists of - (1) data collection; (2) data preprocessing; (3) feature selection/engineering; (4) hyperparameter optimisation and (5) model selection and evaluation.

IoT Datathon 3.0 - #AISAVELIVES
The theme of this datathon was to deliver a machine learning model to predict a dengue outbreak in Singapore, 8 weeks in advance based on the historical data on weather and dengue cases. My goupmates and I created a lightweight time series forecasting regression model based on the XGBoost (eXtreme Gradient Boosting) algorithm to predict the number of dengue cases 8 weeks in advance. Our team was one of the top 8 finalists in this datathon, accredited here.
